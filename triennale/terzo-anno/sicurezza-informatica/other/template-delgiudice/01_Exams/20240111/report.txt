La vulnerabilità di COMMAND INJECTION consiste nello sfruttare la possibilità di iniettare un comando,effettuando una exec() all'interno di una richiesta (come ad esempio in questo caso una GET fatta su di un dominio), 
in modo da poter stampare/leggere dei file interni al web server oppure anche stampare il contenuto di una directory.
Il primo passo è stato quello di trovare il comando giusto per effettuare la exec().
Il primo passo è stato scrivere ?domain=www.ulisse.unibo.it e aggiungere alla fine |, in modo da far partire la exec().
Poi ho provato a trovare il comando giusto per leggere il contenuto del file /tmp/flag_seclab, provando tra head, cat, less, more e, infine, trovando in tail quello giusto.
L'ultimo filtro era relativo al file da leggere e, per scavalcarlo, ho deciso di sostituire la 'b' finale del path con un '?' e di aggiungere ../../ tanti quanti servissero a scavalcare il filtro.
Infine, il comando completo eseguito è stato:

?domain=www.ulisse.unibo.it | tail ./../../../../tmp/flag_secla?

e contenuto del file tmp/flag_seclab:

SEC{Men are born ignorant, not stupid. They are made stupid by education.}
