Vulnerabilità 1 (SQL.Injection)
-Descrizione: La vulnerabilità è stata scoperta durante un tentativo di SQL Injection nella sezione di login del sito.
	Per comodità ho usato il tool Burp per fare la richiesta con i dati modificati(usando + per gli spazi) 
-Payload:
	Username : ' OR '1'='1
	Password : ' OR '1'='1
-Conseguenze: si scatena la query:
	SELECT * FROM users WHERE username='' OR '1'='1' and password='' OR '1'='1'
In questo modo sia le verifiche del nome utente che della password vengano bypassate. 
L'istruzione di cui sopra in realtà interroga tutti gli utenti nel database e quindi ignora la sicurezza.
In questo modo è possibile accedere alle informazioni di tutti gli utenti (carte di credito e documenti vari).
-Bug nel codice: mancanza di validazione dei dati di input utente nel processo di autenticazione, come ad esempio eliminare i caratteri speciali(').

Vulnerabilità 2 (IDOR - Insecure Direct Object Reference)
-Descrizione: La vulnerabilità è stata scoperta nella sezione per visualizzare i dettagli dell'account, dove è presente un caso di Insecure Direct Object Reference (IDOR). 
	È possibile accedere ai dettagli di altri utenti semplicemente cambiando un dettaglio nel link (ad esempio l'ID del file).
-Payload: Non è necessario un payload specifico, ma semplicemente modificare l'ID del file nei parametri dell'URL.
-Conseguenze: Consente agli attaccanti di accedere ai dettagli di altri utenti, violando il principio di accesso sicuro agli oggetti. 
	Ciò potrebbe portare alla divulgazione non autorizzata di informazioni sensibili.
-Bug nel codice: Il bug nel codice deriva dalla mancanza di un adeguato controllo degli accessi agli oggetti. 
	Il sito non verifica correttamente se l'utente ha il permesso di accedere ai dati di un determinato utente prima di fornire le informazioni.

Vulnerabilità 3 (XSS - Cross-site Scripting)
-Descrizione : Dalla barra di ricerca è presenta una vulnerabilità di tipo Cross-site Scripting (XSS) dove è possibile inserire script JS malevoli. 
-Payload : <script>alert(“HiJacked”)</script> 
-Conseguenze : compare come risultato dello script il pop up di alert con la stringa inserita.
Un altro esempio potrebbe essere un payload che reindirizza l'utente a un'altra pagina come una pagina malevola o ad esempio un sito di phishing.
<script>window.location.href="https://evilSite.com"</script>
-Bug nel codice: mancanza di validazione dei dati di input utente nella barra di ricerca. 
	Il sito non filtra correttamente i dati inseriti dall'utente, ad esempio sostituendo la stringa <script>, consentendo l'esecuzione di script JavaScript malevoli.
